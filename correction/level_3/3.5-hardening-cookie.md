# 3.5 — Endurecimento de cookie e defaults de segurança (OPCIONAL)

## Problema
Sem suporte a prefixos `__Host-`/`__Secure-`; `session-id-length=16` (96 bits, conservador);
`ip-hash-salt` default compartilhado e fraco.

## Impacto
Defaults menos seguros que o ideal; risco se o consumidor não sobrescrever o salt.

## Risco
Baixo.

## Arquivos
- `service/SpringSessionLiteCookieManager.java`
- `config/SpringSessionLiteProperties.java`
- `autoconfigure/SpringSessionLiteAutoConfiguration.java` (fail-fast)

## Solução / passos
1. Suporte opcional a prefixo de cookie (`__Host-`/`__Secure-`) — validar requisitos (Secure,
   Path=/, sem Domain para `__Host-`).
2. Subir default de `session-id-length` para 21 (≈128 bits).
3. **Fail-fast no startup:** se `cookie-secure=true` e `ip-hash-salt` ainda for o default
   `spring-session-lite`, logar `WARN` forte ou recusar boot (configurável).

## Complexidade
Baixa

## Verificação
- Teste: salt default + `cookie-secure=true` → WARN/erro no boot.
- Teste: prefixo `__Host-` aplicado respeita restrições.

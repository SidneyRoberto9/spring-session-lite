# 3.2 — Expiração deslizante opcional (OPCIONAL)

## Problema
`last_accessed_at` é atualizado mas `expires_at` nunca estende — sessão morre em TTL fixo mesmo
com uso ativo.

## Impacto
UX: usuário ativo é deslogado abruptamente.

## Risco
Baixo.

## Arquivos
- `service/SpringSessionLiteService.java`
- `config/SpringSessionLiteProperties.java`

## Solução / passos
1. Adicionar propriedade `sliding-expiration` (default `false`).
2. Em `validate`, se ligado, estender `expires_at = agora + ttl` (com **throttle** — reaproveitar
   a lógica de 2.2 para não escrever a cada request).
3. Atualizar `Max-Age` do cookie? (opcional; cookie já tem `Max-Age=ttl`; reescrever Set-Cookie
   no sliding é custo — documentar trade-off).

## Complexidade
Média

## Verificação
- Teste: acesso dentro do TTL com sliding on → `expires_at` estendido (respeitando throttle).
- Sliding off → comportamento atual (TTL fixo).

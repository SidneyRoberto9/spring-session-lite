# 3.3 — Eventos de domínio (OPCIONAL)

## Problema
Sem hooks para auditoria/observabilidade em login/logout/expiração.

## Impacto
Dificulta auditoria, métricas e integrações reativas.

## Risco
Baixo.

## Arquivos
- novos `event/SessionCreatedEvent.java`, `SessionDestroyedEvent.java`, `SessionExpiredCleanupEvent.java`
- `service/SpringSessionLiteService.java` (publicar via `ApplicationEventPublisher`)
- `scheduler/SpringSessionLiteCleanupTask.java`

## Solução / passos
1. Definir eventos imutáveis (`record`) com dados mínimos (userId, sessionId, timestamp — sem PII
   sensível).
2. Injetar `ApplicationEventPublisher` no serviço e publicar em `login`/`logout`/cleanup.
3. Documentar como consumir com `@EventListener`.

## Complexidade
Baixa

## Verificação
- Teste: `@EventListener` recebe `SessionCreatedEvent` no login e `SessionDestroyedEvent` no logout.

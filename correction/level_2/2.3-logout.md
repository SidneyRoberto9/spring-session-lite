# 2.3 — Sem logout / invalidação (e código morto) (IMPORTANTE)

## Problema
`CookieManager.clear()` existe mas **nunca é chamado** (zero refs confirmadas);
`SpringSessionLiteService` não tem `logout`. Impossível invalidar sessão antes da expiração.

## Impacto
Não há revogação (roubo de cookie, troca de senha); recurso esperado ausente; código morto.

## Risco
Médio (segurança/funcional).

## Arquivos
- `service/SpringSessionLiteService.java` (novo método `logout`)
- `service/SpringSessionLiteCookieManager.java` (`clear` já pronto)
- `domain/SpringLiteSessionRepository.java` (delete por sessionId)
- (doc) `docs/01-instalacao-e-uso.md`

## Solução / passos
1. Adicionar `repository.deleteBySessionId(String)` (`@Modifying`).
2. Adicionar `SpringSessionLiteService.logout(HttpServletRequest request, HttpServletResponse response)`:
   - Ler o cookie via mesma lógica do filtro (extrair helper compartilhado p/ ler cookie).
   - `repository.deleteBySessionId(sessionId)`.
   - `cookieManager.clear(response)` (elimina código morto).
3. (Opcional) Sobrecarga `logout(String sessionId)` para uso programático.
4. Documentar endpoint de logout exemplo no consumidor.

## Complexidade
Baixa

## Verificação
- Teste: login → `logout` → cookie de resposta com `Max-Age=0`; linha removida; `/me` → 401.
- `CookieManager.clear` agora referenciado (sem código morto).
- `./mvnw test` verde.

# 2.8 — CORS cross-origin para auth por cookie (IMPORTANTE)

## Problema
A doc mostra `fetch(..., {credentials:"include"})` (UI em outra origem), mas o chain padrão não
configura CORS com `allowCredentials`. Cookie cross-origin exige CORS explícito +
`Access-Control-Allow-Credentials: true` (e origens específicas, não `*`).

## Impacto
Integração UI↔API em origens distintas (ex.: Next.js + Spring) falha silenciosamente.

## Risco
Médio (integração/DX).

## Arquivos
- `autoconfigure/SpringSessionLiteAutoConfiguration.java`
- `config/SpringSessionLiteProperties.java` (novas propriedades)
- (doc) `docs/01` / `docs/02`

## Solução / passos
1. Adicionar propriedades: `cors-enabled` (default `false`), `cors-allowed-origins`
   (`List<String>`), `cors-allowed-methods`, `cors-allow-credentials` (default `true`).
2. Quando `cors-enabled=true`, aplicar `CorsConfigurationSource` no chain padrão
   (`http.cors(...)`) com `allowCredentials=true` e origens explícitas (rejeitar `*` + credentials).
3. Documentar configuração e o lado UI (`withCredentials`/`credentials:"include"`).
4. Para apps com chain próprio, documentar como reaproveitar o `CorsConfigurationSource` exposto.

## Complexidade
Média

## Verificação
- Teste: preflight `OPTIONS` de origem permitida → headers CORS com credentials.
- Origem não listada → bloqueada.
- `./mvnw test` verde.
